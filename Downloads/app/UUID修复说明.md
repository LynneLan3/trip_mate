# 🔧 UUID 修复说明

## ❌ 原因

之前的 SQL 脚本使用了字符串作为问卷 ID（如 `'dest_preference'`），但 Supabase 数据库中的 `quizzes` 表的 `id` 字段是 **UUID 类型**，导致导入失败。

错误信息：
```
ERROR: 22P02: invalid input syntax for type uuid: "dest_preference"
```

## ✅ 已修复

现在所有的问卷 ID 都使用标准 UUID 格式：

| 问卷 | UUID |
|------|------|
| 目的地偏好测试 | `a0000000-0000-0000-0000-000000000001` |
| 餐饮习惯测试 | `a0000000-0000-0000-0000-000000000002` |
| 消费习惯测试 | `a0000000-0000-0000-0000-000000000003` |
| 住宿偏好测试 | `a0000000-0000-0000-0000-000000000004` |
| 景点偏好测试 | `a0000000-0000-0000-0000-000000000005` |

## 🚀 现在可以导入了

1. 打开 Supabase SQL Editor
2. 复制修复后的 `新增5个问卷类型.sql` 全部内容
3. 粘贴并执行
4. 等待成功提示

## 📝 修改内容

- ✅ 所有 `quizzes` 表的 INSERT 语句：ID 改为 UUID 格式
- ✅ 所有 `quiz_questions` 表的 INSERT 语句：`quiz_id` 改为对应的 UUID
- ✅ 验证查询的 WHERE 条件：改为 UUID 列表

## 🎉 修复完成

SQL 脚本已经完全修复，可以正常导入到数据库中了！
