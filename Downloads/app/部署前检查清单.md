# 📋 部署前检查清单

## ✅ 必须完成的配置

### 1. Git 仓库
- [ ] 项目已推送到 GitHub/GitLab/Bitbucket
- [ ] main/master 分支是最新的代码
- [ ] 包含所有必要文件（package.json, src/, vercel.json等）

### 2. 环境变量
- [ ] 在 Vercel 项目设置中添加：
  ```
  VITE_SUPABASE_URL=https://your-project.supabase.co
  VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
  ```
- [ ] 变量名称正确（带 VITE_ 前缀）

### 3. Supabase 配置
- [ ] 数据库已启用
- [ ] RLS 权限已正确配置
- [ ] 存储桶权限已设置（如果使用头像上传）

### 4. 构建测试
运行以下命令确保本地构建正常：
```bash
npm run build
```
- [ ] 构建成功，无错误
- [ ] dist/ 目录已生成

## 🔧 可选优化

### 性能优化
- [ ] 图片已优化

- [ ] 移除不必要的依赖

### SEO 优化
- [ ] 页面标题已设置

- [ ] 网站图标已添加

## 🚀 部署步骤

1. [ ] 打开 [Vercel Dashboard](https://vercel.com/dashboard)
2. [ ] 点击 "Add New..." → "Project"
3. [ ] 选择你的 Git 仓库
4. [ ] 配置构建设置：
   - Framework: Vite
   - Root Directory: ./
   - Build Command: npm run build
   - Output Directory: dist
5. [ ] 添加环境变量
6. [ ] 点击 "Deploy"

## 🔍 故障排除

### 如果构建失败
1. 检查 package.json 中的依赖版本
2. 确保 Node.js 版本兼容（推荐 18.x 或 20.x）
3. 查看 Vercel 构建日志中的具体错误

### 如果功能异常
1. 检查环境变量是否正确设置
2. 验证 Supabase 连接
3. 查看浏览器控制台错误

### 如果性能问题
1. 启用 Vercel Analytics
2. 检查图片和资源大小
3. 考虑代码分割优化
